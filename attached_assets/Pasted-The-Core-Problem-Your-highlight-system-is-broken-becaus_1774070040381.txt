The Core Problem
Your highlight system is broken because it's tied to TTS audio progress as a single linear number (ttsProgress = 0→1), but each slide has different numbers of elements to highlight, and the script doesn't spend equal time on each data point. The getHighlightStep function just divides progress evenly — so if Slide 2 has 12 highlight steps but the narrator spends 70% of the time on the first 3 data points, the highlights race ahead or lag behind.

Here's what needs to change:

1. Time-stamped highlight cues per slide (not linear progress)
Each script mentions specific data points at specific moments. You need a HIGHLIGHT_CUES array that maps percentage of audio progress → which highlight step to be on, so highlights sync to what's actually being spoken.

2. The hlStep function needs to use cue-based lookup
Instead of Math.floor(progress * totalSteps), it should find the correct step from the cue timeline.

3. SectionDataCards auto-expand timing is disconnected
The EXPAND_AT_STEP map (step 6 = "sold-for", step 7 = "sold-to", etc.) works for the timer fallback but doesn't sync with audio narration progress properly.

function getHighlightStep(progress: number, cues: [number, number][]): number {

  }
